## 2. ストーリー（Why）

### 2.1 背景

- Backend は `virtual-voicebot-backend/docs/` に体系的な SoT ドキュメント（RD/BD/DD/UT/ST）+ ステアリング運用が整備済み
- Frontend のドキュメント（RD-005, BD-001）はルート `docs/` に散在し、テンプレート・インデックス・プロセス定義が未整備
- Backend と同じ V 字トレースが Frontend では成立しておらず、AI エージェント（Codex/CodeRabbit）が参照する仕様が不足

### 2.2 目的

- Frontend 専用の SoT ドキュメント体系を `virtual-voicebot-frontend/docs/` に構築する
- Backend と対称のディレクトリ構造・テンプレート・プロセスを整備する
- Frontend 技術スタック（Next.js / React / TypeScript）に適合したテンプレートを用意する

### 5.3 Backend との差異

| 観点 | Backend | Frontend |
|------|---------|----------|
| 言語 | Rust | TypeScript |
| DD テンプレート | `fn xxx() -> Result<>` | `function xxx(): ReturnType` / React Component |
| UT テンプレート | `cargo test` | Vitest / React Testing Library |
| 品質ゲート（静的解析） | `cargo clippy` / `cargo fmt` | ESLint / Prettier |
| 品質ゲート（ビルド） | `cargo build` | `pnpm build` (Next.js) |
| DD の追加観点 | 状態遷移 | コンポーネント Props / Hooks / 状態管理 |
