# 生成AIにるソフトウェアの設計

## 設計工程へのAI適用方針
- ソフトウェア設計とは
    - 「何を解くか」を定めた要求仕様をもとに、「どのようにして解くか」のレベルまで具体化し、実装工程へのスムーズな移行を目指す・
- 設計文章
    - 設計判断の経緯や結果をステークスホルダに伝えるためのコミュニケーション媒体。
        - 図、表、自然言語、数理的な形式記述など。
## 生成AI適用の課題と方針
- 設計工程においては、人間が主な作業者となり、一部の小さな作業を生成AIに代替・支援させるような適用方針がふさわしい。
    - 文献も非常に少ない
1. 支援、代替させる作業に必要な文脈情報を
2. 他の不要な情報を省きつつ
3. いかに正確に理解させるか

## 設計書からの情報生成
- 設計書の読み込み
生成AIに、より上位文書を読み込ませる必要がある。
    - 読み込み：上位文書の読み込ませる際に、後続タスクの必要な程度、生成AIが記述内容を理解できるように加工・修正して与えること。
    - 生成AIに読みやすい設計書
        - 人間がチェックしやすい設計書であることも兼ねる
    - テキストマークアップ言語
        - 生成AIが学習済みの機能(XMLなどのケースでは)を用いる
        - 図をマークアップで記述する
            - PlantUML
            - Marmaid形式
        - 表：行、列の見出しがどこか、データ部分がどこかわかりやすくする
        - DB設計でも意識する
        - 表データの「テキストとしての自然さ」は、続く応答の品質に影響する。
            - 人間が理解できないような入力まではしなくてよい、読み込み時に前処理などを行うと良い

     - 保守、流用開発で既存の設計文書を読むとき
