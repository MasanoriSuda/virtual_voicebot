## 概要

本ディレクトリには、システムの要件仕様書（RD: Requirements Definition）を格納する。

- 各 RD は「**何を作るか**」を定義する
- 対応するテスト：システムテスト（ST）
- トレーサビリティ：RD ↔ ST

---

## 命名規則

```
RD-{連番}_{slug}.md

例:
- RD-001_call-reception.md
- RD-002_ai-conversation.md
- RD-003_recording.md
```
